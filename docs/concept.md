# Yori（より）- サービスコンセプトドキュメント

## サービス概要

**サービス名**: Yori（より）
**サブコピー**: 「寄り添い」の寄り、「より良く」のより

### ひとことで言うと
障害のある子を育てる親が、今日のしんどさや嬉しさをAIに話す。言葉になったら、専門家や支援者に渡せる形になる。

---

## 解決する課題

### ターゲットユーザー
- 障害のある子ども（主に自閉スペクトラム傾向、発達障害など）を育てる親
- ママ・パパどちらも対象。孤独を感じやすい点は共通している

### 課題の本質
- しんどさは言語化できていない、ではなく「**言語化する機会がなかっただけ**」
- 話す場所はある（コミュニティ、ケースワーカー）でも「**一人では踏み出せない**」
- 専門家の存在は知っている、でも「**うまく話せるか不安で繋がれない**」
- 自分のためにケアを受けることへの抵抗がある

### 日常にある感情の断面（ユーザーインタビューより）
- 子どもが崩れた後、一人で立て直してそのまま誰にも言えなかった
- きょうだい児を後回しにしてしまった罪悪感
- 配偶者との温度差を感じた夜
- また怒鳴ってしまった直後の自己嫌悪（「またやってしまった」という繰り返し感）
- ずっとできなかったことが今日はじめてできた喜びを、誰かに話したかった

---

## サービスの体験設計

### 入口の設計（重要）
「あなたのために」ではなく**「子どものことを話す場所」** として入口を設計する。
話しているうちに、自分のしんどさも嬉しさも自然に出てくる流れにする。

### 体験の流れ

```
0. AIから「今日どうでしたか？」と話しかけてくれる
   ↓
1. 子どものことを話す（入口）
   ↓
2. 話すうちに自分の感情が出てくる
   （しんどさも、小さな喜びも）
   ↓
3. 感情が言語化・ログ化される
   （例：「また怒鳴ってしまった」「でも今日はじめてスプーンを持てた」）
   ↓
4. 積み重なったログを持って専門家に相談できる
   ↓
5. 相談先をサジェスト（将来的にはマッチング）
```

### なぜAIか
- 評価されない
- 気を遣わなくていい
- 何時でもいい
- 人には話せないことが話せる
- 弱さを見せることへのハードルが低い

## MVP機能定義（現在の実装）

### オンボーディング
初回ログイン時に以下を登録:
- パパ / ママ の選択
- 子どもの情報（ニックネーム任意・生年月日・性別）
- 複数の子どもに対応（「＋もう一人追加」）

Yoriはこの情報をもとに、`お母さん` / `お父さん` と呼びかけ、子どもの年齢・性別を文脈として持ちながら話す。登録内容はアカウントページからいつでも編集できる。

### 機能1: AIとの対話（チャット）
**最も重要なコア機能**

- AIが毎日話しかけてくれる（「今日はどんな一日でしたか？」）
- しんどさも、小さな成長の喜びも、どちらも同じように受け止める
- 判断・評価をしない
- 24時間いつでも使える
- 「今日の話を終える」で会話を締めくくり、ねぎらいメッセージを受け取る
- 「また話す」で同日中に新しい会話を始められる
- 「今日の話を終える」を押さなくても、0時に自動で記録が保存される
- 地域の支援サービス・施設・制度について質問するとWeb検索して情報を案内する

**Yoriの応答設計：状態に応じてモードを切り替える**

Yoriは会話の流れからユーザーの状態を読み取り、応答スタイルを変える。一辺倒な共感ではなく、そのときに必要なものを返す。

| モード | 判断の目安 | Yoriの応答 |
|--------|-----------|-----------|
| 感情放出 | 強い感情・疲れ・余裕のなさ | 共感のみ。質問・アドバイスなし。「そのままでいい」を伝える |
| 内省 | 落ち着いて振り返りたそう | 共感＋出来事に軽く触れる。必要なら1問だけ |
| 改善 | どうすればいいか考えたそう | 共感＋軽い整理・小さな提案 |
| 喜び | 子どもの成長・嬉しい出来事 | 一緒に喜ぶ。積み重ねを認める |

**質問のルール**
- 感情放出モードでは絶対にしない
- 1回の返答に1つまで
- 感情を詰めるのではなく「出来事・場面」を聞く

### 機能2: 記録・ログ機能
**対話から自動生成される**

- 対話内容から出来事・気持ち・成長を自動抽出・保存
- ユーザーが意識して記録しなくていい（話すだけで溜まる）
- 記録一覧・詳細で振り返れる
- 詳細画面からそのときのチャット履歴も読み返せる
- 1セッション = 1記録。同じ日に複数回話した場合は「1回目」「2回目」として別々に保存される

**ログの構造**
```
日付（+ 同日複数の場合は回数）
├── 出来事（事実）: 「次男が発熱、長男が春休みで在宅」
├── 気持ち（感情）: 「地獄だった、誰にも頼れなかった」
├── できたこと・成長: 「はじめてスプーンを自分で持てた」（あれば）
└── タグ: [きょうだい, 体調, ワンオペ, はじめてできた]
```

### 機能3: 相談先サジェスト（簡易版）
- ログの内容から「今の状態に合った相談先」を提示
- 「この内容を持って、専門家に相談してみませんか」という背中押し
- 将来的にはマッチング機能へ

## 画面構成（現在の実装）

### 1. ログイン画面（`/`）
- Googleログインボタンのみ
- コピー：「今日のしんどさを、話してみませんか。」

### 2. オンボーディング画面（`/onboarding`）
- 初回のみ表示（`profiles.parent_type` が未設定の場合）
- パパ/ママ選択 + 子ども情報入力

### 3. ホーム画面（`/home`）
- Yoriからの話しかけカード
- 「話す」ボタン
- 今週の記録一覧
- ナビの「アカウント」からプロフィール編集・ログアウトへ

### 4. チャット画面（`/chat`）
- メインの対話UI（SSEストリーミング）
- 「今日の話を終える」→ ねぎらいメッセージ + 記録保存 + 「また話す」

### 5. 記録一覧（`/logs`）
- 日付・出来事・気持ち or 成長 + タグ
- 同日に複数の記録がある場合は「1回目」「2回目」のラベルを表示

### 6. 記録詳細（`/logs/[id]`）
- 出来事 / 気持ち / できたこと・成長 / タグ
- そのときの会話履歴

### 7. アカウント（`/account`）
- プロフィール編集（パパ/ママ・子ども情報）
- ログアウト
- プライバシーポリシー・お問い合わせへのリンク
- 退会（2ステップ確認）

## このドキュメントの背景

このコンセプトは、障害のある子どもを育てる当事者との対話から生まれました。

「ビジネスの相談」として始まった会話の中で、当事者自身のしんどさが自然に言語化され、「誰にも話せていなかった」という孤独が明らかになりました。

その体験そのものが、Yoriの体験設計の原型です。

**最初のユーザーは、このサービスを作っている人自身。**

発達障害の子どもは、遅れて「できること」が増えていく。しんどさと一緒に、その喜びもちゃんと記録に残したい。そういう場所を作りたい。

---

## 認証設計

**方式: Googleログイン（Supabase Auth + Google OAuth 2.0）**

### 選定理由
- ボタン一つで完結、フォーム入力不要
- 「今日しんどい」瞬間の離脱を防ぐ
- 「本人しか見れない」という安心感を担保する

### 画面フロー
```
ランディング（未ログイン）
  ↓「Googleでログイン」ボタン
Google認証
  ↓ 認証成功
オンボーディング（初回のみ）
  ↓ 設定完了
ホーム画面
```
